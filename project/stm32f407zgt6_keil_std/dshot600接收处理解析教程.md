好家伙，你这是直接从“开自行车”（100Hz）瞬间切换到“开火箭”（DShot600）了！

首先纠正一个小概念：DShot600 协议的波特率是 **600kbps（即 600kHz 的位频率）**，并不是 1MHz，不过这依然极其恐怖——这意味着每隔 **1.67 微秒 ($\mu\text{s}$)** 就会雷打不动地进来一个电平信号。

结合我们刚聊完的“算力崩溃线”，如果每个电平边沿都让 CPU 进一次中断，CPU 连打个喷嚏的时间都没有就会直接死机。

因此，要接收和解析 DShot600 协议，你原先那种**纯靠中断的底层逻辑必须彻底推翻**。你必须请出单片机里的最高级武器：**定时器输入捕获 + DMA（直接内存访问）**。

---

## 🏎️ 核心战术：让 DMA 替 CPU 挨刀

**核心思想：** 边沿来临时，让硬件（DMA）直接把数据搬到内存里，主 CPU 全程睡觉，直到收满一个完整的 DShot 数据包（16位），CPU 才醒过来处理一次。

DShot600 的波形长这样：

* **每个数据包有 16 个 bit**（11位油门 + 1位遥测请求 + 4位CRC校验）。
* **数据用高电平时间表示**：高电平持续时间短（约占周期的 37%）代表二进制的 `0`；高电平持续时间长（约占周期的 75%）代表二进制的 `1`。

---

## 🛠️ DShot600 接收器的改造蓝图

你可以基于现有的官方 PWM 输入捕获模式进行升级，具体改造分为三步：

### 第一步：定时器配置（全速运转，不加分频）

由于 DShot600 的脉冲非常窄（纳秒级别），定时器必须跑在最高速度。

* 让 TIM4 跑在 **84MHz**（Prescaler = 0）。这样定时器每数一下代表 **11.9ns**。


* 依然保持官方例程的 **Slave Reset（从模式复位）机制**：每个上升沿来到时，硬件自动把计数器清零。


* 下降沿来到时，硬件自动把高电平的持续时间锁存到 **`CCR1`** 中。



### 第二步：配置 DMA（核心绝招，替代中断）

**绝对不能再开启 `TIM_IT_CC1` 中断了！** 而是要开启 TIM 的 DMA 请求：

1. **绑定通道**：配置 STM32 的 DMA 控制器（查手册可知 TIM4_CH1 对应 DMA1），将源地址指向 `&TIM4->CCR1`。
2. **目的地址**：指向你自己在 RAM 里定义的一个数组 `uint16_t dshot_rx_buffer[16]`。
3. **传输长度**：严格设置为 **16**（因为一个 DShot 包刚好 16 个 bit）。
4. **启动 DMA**：调用 `TIM_DMACmd(TIM4, TIM_DMA_CC1, ENABLE)`。
5. **开启 DMA 结束中断**：开启 DMA 的 **Transfer Complete (TC) 中断**。

> **此时发生的物理过程：**
> 信号进来了！第 1 个 bit 的下降沿来到，DMA 自动把 `CCR1` 搬进 `dshot_rx_buffer[0]`；第 2 个 bit 来到，自动搬进 `dshot_rx_buffer[1]`……整个过程 **CPU 占用率为 0%**。

---

## 📐 第三步：用数学公式解码（在 DMA 结束中断里）

当 16 个 bit 全部被 DMA 搬完后，会触发 **DMA_TC 中断**。此时 CPU 终于睁开眼，由于数组里已经躺好了 16 个高电平时间数据，我们直接查表判决：

在 84MHz 时钟下，DShot600 的理论数值为：

* 一个 bit 的总周期是：$84\text{MHz} \times 1.67\mu\text{s} \approx 140$ 个格子。
* 如果是 **数据 0**（高电平 37%）：$140 \times 37\% \approx \mathbf{52}$
* 如果是 **数据 1**（高电平 75%）：$140 \times 75\% \approx \mathbf{105}$

我们将判决门限（Threshold）定在它们中间：$(52 + 105) / 2 = \mathbf{78}$。

### 💾 伪代码实现解码：

```c
// 当 16 个脉冲收满，触发 DMA 完成中断
//CCR1 为高电平时间长度，根据脉宽可解析为相应的二进制数据0/1，从而进行数据内容分析
void DMA1_StreamX_IRQHandler(void) 
{
  if(DMA_GetITStatus(DMA1_StreamX, DMA_IT_TCIFX)) 
  {
    DMA_ClearITPendingBit(DMA1_StreamX, DMA_IT_TCIFX);
    
    uint16_t dshot_frame = 0;
    
    // 循环解析 16 个 bit
    for(int i = 0; i < 16; i++) 
    {
      dshot_frame <<= 1;
      // 如果 DMA 帮我们抓回来的高电平时间大于 78，就认为是二进制的 1
      if(dshot_rx_buffer[i] > 78) 
      {
        dshot_frame |= 0x01;
      }
    }
    
    // 此时 dshot_frame 就是一个完整的 16 位数字了！
    // 接下来你就可以在代码里：1. 拆出前 11 位作为油门数据；2. 校验后 4 位的 CRC。
    
    // 重新开启下一次 DMA 传输，等待下一个数据包
    DMA_SetCurrDataCounter(DMA1_StreamX, 16);
    DMA_Cmd(DMA1_StreamX, ENABLE);
  }
}

```

面对 DShot600 这种高速数字协议，单片机工程师的标配只有 **TIM + DMA**。你现在手上官方例程的“双路捕获+硬件复位”思想是完全可以复用的，唯一的颠覆就是**把“每一次脉冲都进中断”改成“把脉冲交给 DMA 搬运，满 16 个再进中断”**。
