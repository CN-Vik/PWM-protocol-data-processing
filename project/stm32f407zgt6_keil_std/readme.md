# 一、这个例子到底想干什么？

一句话：

> **测量外部PWM信号的周期和占空比。**

例如外部来了一个PWM：

```
____████________████________████____

<------20us------>

<--8us-->
```

它要测出来

```
周期 = 20us

高电平 = 8us

占空比 = 8 / 20 = 40%
```

就这么简单。

>

>

> 占空比

所以代码非常少。

# 十一、这个例子为什么特别适合 DShot600？

你之前说你的目标是：

> **检测 PWM 的占空比，周期其实不太关心，占空比代表二进制 0/1。**

对于 DShot600：

* 位周期固定约 **1.67 μs**（600 kbit/s）。
* 逻辑 **1**：高电平约 **1.25 μs**（约 75% 占空比）。
* 逻辑 **0**：高电平约 **0.625 μs**（约 37.5% 占空比）。

因此，每收到一个 PWM 位，你只需要读取：

* `CCR1`：高电平时间；
* `CCR2`：整个位周期（可用于校验时序是否正常）。

然后计算：

```c
duty = (float)CCR1 / CCR2;
```

再设置一个阈值，例如：

```c
if (duty > 0.56f)
{
    bit = 1;
}
else
{
    bit = 0;
}
```

这正是 DShot 接收最常见的判决思路之一。

不过需要注意一个关键点：**官方这个 `TIM_PWMInput` 示例本身更适合测量连续、稳定周期的 PWM 信号**（例如舵机 PWM、风扇测速 PWM 等）。而 **DShot 是一串连续的数据脉冲**，每个 bit 都会产生新的捕获结果，因此在实际工程中通常还需要结合 **DMA**（自动保存每次捕获值）或更高效的中断处理，否则仅靠这个示例很容易跟不上 DShot600 的数据速率。

所以，这个官方例子的价值主要在于帮助你彻底理解 **PWM 输入捕获的硬件工作原理**。理解了它之后，再去看 STM32 的 **TIM Input Capture + DMA** 示例，你就会发现 DShot 接收实际上就是把这里的捕获结果连续存储下来，然后再软件解析成 16 位数据。

# 结合内部硬件再理解一次（最重要）

下面这个图，是官方 **PWM Input Mode** 的完整工作过程，也是这个例子的本质。

```
              PB7(TIM4_CH2)
                    │
                    │
          外部PWM输入信号
                    │
                    ▼
             TI2FP2（滤波后输入）
                    │
          ┌─────────┴─────────┐
          │                   │
          ▼                   ▼
     CH2 捕获上升沿        CH1 捕获下降沿
          │                   │
          │                   │
      CCR2 = 周期         CCR1 = 高电平
          │
          ▼
   上升沿触发 Reset Mode
      CNT 自动清零
          │
          ▼
     CC2 捕获完成中断
          │
          ▼
 TIM4_IRQHandler()
          │
          ├── 读取 CCR1（高电平）
          ├── 读取 CCR2（周期）
          ├── Duty = CCR1 / CCR2
          └── Freq = TIMCLK / CCR2
```

整个硬件实际上已经帮你完成了**高电平宽度测量、周期测量、计数器清零和同步**，CPU 在中断里只是**读取结果并计算**。

## 最后结合你的 DShot600 目标说一句

你之前一直在找**哪个官方例子最适合做 DShot600 接收**。

答案就是：**这个例子就是理解 DShot 接收的第一步**。

但是实际的 DShot600 接收一般不会在中断里像这样只读一次 `CCR1` 和 `CCR2`，而是会改成：

```
每次捕获
    ↓
DMA 自动把 CCR 值存到数组
    ↓
收到一整帧（16 位）
    ↓
CPU 再统一解析每个 bit 的高电平时间
    ↓
判断是 0 还是 1
```

所以可以说：

* **这个官方例子**：教你 **PWM Input Mode 的硬件工作原理**；
* **下一步学习**：就是 **Input Capture + DMA**，这是 STM32 接收 DShot600 最常见、也是效率最高的方案。
